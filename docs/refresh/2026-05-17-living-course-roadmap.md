# Living Course Roadmap

Date: 2026-05-17
Project: Codex In Plain English

## Position

Treat this as a living project. Codex is moving toward a broader work app, so the course should keep expanding as new capabilities change how people delegate, review, or repeat digital work.

The course should not become developer documentation. It should translate current Codex behavior into plain-English operating choices.

## Course Architecture

### Core Course

Modules 1-8 remain the original foundation. They are marked `Updated May 17` when source attribution, terminology, or current-product context changes.

### Expanded Codex 2026

Modules 9-14 explain what changed since the original course:

- Codex as a broader work surface.
- In-app browser, Chrome, and Computer Use.
- Mobile remote control and steering.
- Durable threads, memory, and work notes.
- Goals, Heartbeats, and long-running loops.
- Review, safety, and the living-course update pattern.

## Update Rules

Add a new module when a Codex feature changes the learner operating model.

Update an existing module when a feature clarifies or extends a concept already taught there.

Add a source note when a new article, official doc, or practitioner example informs the course.

Mark learner-facing modules as `New` or `Updated` so returning learners can jump directly to what changed.

## Source Rules

Use official OpenAI Codex docs for current product behavior.

Use practitioner writing for workflow framing and examples.

Keep Gabriel Chua's plain-English delegation framing visible as the original inspiration.

Use Markdown source notes for articles, docs, and refresh decisions so the course has an inspectable history.
